У кортежей (**tuple**) в Python методов **гораздо меньше**, чем у списков. Это связано с тем, что кортежи — **неизменяемые** (immutable) последовательности. Их нельзя изменить после создания: нельзя добавлять, удалять или заменять элементы.

## Все методы кортежей (всего 2)

Так как кортеж неизменяем, у него есть только **методы поиска и подсчета** (которые не меняют объект):

### 1. **`.count(x)`** — возвращает количество вхождений элемента `x`

```python
t = (1, 2, 2, 3, 2)
c = t.count(2)      # c = 3
c = t.count(5)      # c = 0 (нет ошибки, просто 0)
```

### 2. **`.index(x, start, end)`** — возвращает индекс **первого** вхождения `x`

```python
t = (10, 20, 30, 20, 40)
i = t.index(20)     # i = 1 (первое вхождение)
i = t.index(20, 2)  # i = 3 (ищем с индекса 2)
```

Если элемент не найден — вызывает `ValueError`:
```python
t = (1, 2, 3)
t.index(5)  # ValueError: tuple.index(x): x not in tuple
```

## Чего **НЕТ** у кортежей (в отличие от списков)

Следующие методы **отсутствуют**, потому что они изменяют объект:
- ❌ `.append()` 
- ❌ `.extend()`
- ❌ `.insert()`
- ❌ `.remove()`
- ❌ `.pop()`
- ❌ `.clear()`
- ❌ `.sort()`
- ❌ `.reverse()`
- ❌ `.copy()` (но скопировать можно через `tuple()` или срез)

## Встроенные функции, работающие с кортежами

Хотя это не методы, они часто используются с кортежами и возвращают новые объекты:

```python
t = (3, 1, 4, 1, 5)

len(t)          # 5 - длина
sorted(t)       # [1, 1, 3, 4, 5] - возвращает список! (не кортеж)
min(t)          # 1
max(t)          # 5
sum(t)          # 14
tuple(sorted(t)) # (1, 1, 3, 4, 5) - если нужен кортеж
```

## Сравнение: список vs кортеж

| Операция | Список (list) | Кортеж (tuple) |
|----------|---------------|----------------|
| Изменяемость | ✅ Да | ❌ Нет |
| `.append()` | ✅ | ❌ |
| `.insert()` | ✅ | ❌ |
| `.remove()` | ✅ | ❌ |
| `.pop()` | ✅ | ❌ |
| `.sort()` | ✅ | ❌ |
| `.reverse()` | ✅ | ❌ |
| `.clear()` | ✅ | ❌ |
| `.copy()` | ✅ | ❌ |
| `.count()` | ✅ | ✅ |
| `.index()` | ✅ | ✅ |
| `len()` | ✅ | ✅ |

## Зачем нужны кортежи, если у них мало методов?

1. **Быстрее** списков (меньше накладных расходов)
2. **Безопаснее** — данные защищены от случайного изменения
3. **Могут быть ключами словаря** (списки — нет)
4. **Лучше для неизменяемых данных** (координаты, RGB-цвета, даты)

Пример использования:
```python
# Кортеж как ключ словаря
coordinates = {(10, 20): "A", (30, 40): "B"}  # ✅ работает

# Список как ключ - ОШИБКА
# coordinates = {[10, 20]: "A"}  # TypeError: unhashable type: 'list'
```
