В объектно-ориентированном программировании (ООП) в Python объекты представляют собой сущности, которые объединяют в себе **данные** (состояние) и **поведение** (операции над этими данными). Это ключевая идея: объект — это не просто набор переменных, а целостная модель, которая умеет что-то делать со своим состоянием.

### Данные (атрибуты)

Данные объекта хранятся в **атрибутах**. Их можно разделить на:

- **Атрибуты экземпляра** — принадлежат конкретному объекту и определяются в методе `__init__` (конструкторе). Они задают состояние объекта.
- **Атрибуты класса** — общие для всех экземпляров класса. Их изменение влияет на все объекты (если не переопределены в экземпляре).

Пример:
```python
class Cat:
    species = "Felis catus"  # атрибут класса

    def __init__(self, name, age):
        self.name = name      # атрибут экземпляра
        self.age = age        # атрибут экземпляра
```

Здесь у каждого кота будут свои `name` и `age`, но все коты разделяют общий `species`.

### Поведение (методы)

Поведение объекта реализуется через **методы** — функции, определённые внутри класса. Они могут работать с данными объекта. Основные виды методов:

- **Методы экземпляра** — принимают `self` (ссылку на текущий объект) и могут читать или изменять его атрибуты.
- **Методы класса** (`@classmethod`) — принимают `cls` (ссылку на класс) и работают с атрибутами класса.
- **Статические методы** (`@staticmethod`) — не получают ни `self`, ни `cls`, это просто функции, связанные с классом по смыслу.

Пример:
```python
class Cat:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def meow(self):          # метод экземпляра
        print(f"{self.name} says Meow!")

    @classmethod
    def get_species(cls):
        return cls.species

    @staticmethod
    def is_adult(age):
        return age >= 2
```

### Как данные и поведение связаны?

В методах экземпляра через параметр `self` мы обращаемся к атрибутам конкретного объекта. Это позволяет методу вести себя по-разному для разных объектов. Например:

```python
fluffy = Cat("Fluffy", 3)
tom = Cat("Tom", 1)
fluffy.meow()  # Fluffy says Meow!
tom.meow()     # Tom says Meow!
```

Метод `meow` использует атрибут `name` того объекта, для которого он вызван.

### Инкапсуляция

В Python нет строгой приватности, но есть соглашение: атрибуты и методы, начинающиеся с подчёркивания (`_`), считаются защищёнными, а с двойным подчёркиванием (`__`) — имя искажается (name mangling), чтобы избежать случайных переопределений. Для управления доступом к данным используют **свойства** (`@property`), которые позволяют добавить логику при получении или изменении атрибута, сохраняя интерфейс обращения как к обычному полю.

```python
class Cat:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        if value:
            self._name = value
        else:
            raise ValueError("Name cannot be empty")
```

Здесь данные (`_name`) защищены, а поведение (геттер и сеттер) контролирует доступ к ним.

### Итог

- **Данные** — это атрибуты, хранящие состояние объекта.
- **Поведение** — это методы, определяющие, что объект может делать со своими данными.
- Вместе они образуют единую сущность — объект, что делает код более организованным, переиспользуемым и соответствующим реальному миру.

В Python это реализовано гибко, с возможностью управлять доступом и расширять поведение через наследование и полиморфизм.
