# T-Invest API to Postgres ETL Pipeline

Учебный ETL-пайплайн для загрузки биржевых свечей из API T-Invest в PostgreSQL.

## Текущий статус
Проект в активной разработке. В процессе реализации модульная ООП-архитектура, добавлен Airflow для оркестрации, этап настройки пайплайна и подготовки к расширению.

## Что делает
Стягивает свечи по заданному тикеру и сохраняет в БД. Если запустить повторно — новые данные добавятся, старые не перезатрутся (инкрементальная загрузка).

## Технологии
- Python (requests, pandas, sqlalchemy)
- PostgreSQL (в Docker)
- API T-Invest
- Airflow (в Docker)

## Структура
```
├── main.py                 # точка входа для ручного запуска
├── src/
│   ├── api_client.py       # работа с T-Invest API
│   ├── db_manager.py       # работа с PostgreSQL
│   └── etl_pipeline.py     # ETL-логика
├── airflow/
│   ├── dags/               # DAG-файлы Airflow
│   ├── logs/               # логи Airflow
│   └── plugins/            # плагины Airflow
├── dev/                    # экспериментальные ноутбуки
│   ├── get_assets.ipynb
│   └── get_currencies.ipynb
├── docker-compose.yml      # теперь с Postgres + Airflow
├── .env.example
├── .env
├── sql.ipynb
└── README.md
```

## Как запустить

1. Скопируй и заполни `.env`:
```bash
cp .env.example .env
# открой .env и впиши свой T_TOKEN
```

2. Создай необходимые папки для Airflow:
```bash
mkdir -p ./airflow/dags ./airflow/logs ./airflow/plugins
```

3. Подними все сервисы через Docker Compose:
```bash
docker-compose up -d
```
Теперь поднимаются одновременно PostgreSQL и Airflow.

4. Запусти пайплайн:
- **Вручную** — выполни:
```bash
python main.py --figi BBG004730N88 --interval 15min --days 1
```

## 2026-06-21 Что изменилось

**Полный рефакторинг в ООП-стиль.** Вместо одного скрипта `get_candles.py` — модульная архитектура с разделением ответственности:

- `src/api_client.py` — класс `TInvestClient`: работа с API T-Invest (запросы, обработка ответов, CamelCase → snake_case).
- `src/db_manager.py` — класс `DatabaseManager`: подключение к PostgreSQL, создание таблиц, проверка дубликатов, сохранение данных.
- `src/etl_pipeline.py` — класс `CandlesETL`: оркестрирует весь процесс загрузки (extract → transform → load).
- `main.py` — точка входа с аргументами командной строки (`--figi`, `--interval`, `--days`).

**Что улучшилось:**
- Код стал читаемым, переиспользуемым и готовым к расширению.
- Появилось логирование на каждом этапе — видно, что происходит.
- Добавлена проверка дубликатов перед вставкой.
- Все подключения к БД и API вынесены в переменные окружения (`.env`).
- Аргументы командной строки — можно гибко запускать для разных инструментов и периодов.

**Как запустить:**
```bash
python main.py --figi BBG004730N88 --interval 15min --days 1
```

**Что дальше:** Архитектура готова к расширению. В планах добавить загрузку других сущностей (акции, валюты) через ту же структуру — просто создавать новые ETL-классы.
